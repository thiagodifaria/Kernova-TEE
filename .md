Um ecossistema de segurança de nível de kernel projetado para criar uma zona de confiança absoluta (Trusted Execution Environment) através de um Micro-Hypervisor de Tipo-1. Ele resolve o problema de latência e segurança em operações criptográficas críticas, protegendo dados contra rootkits e vazamentos de memória no sistema operacional host.

Finalidade e Objetivos
A finalidade é fornecer uma camada de abstração de hardware que isola processos sensíveis sem a sobrecarga de uma máquina virtual completa.

Isolamento Físico: Utilizar virtualização assistida por hardware (Intel VT-x) para separar o contexto de segurança do restante do SO.

Performance Incomparável: Mover dados entre zonas de segurança com latência próxima de zero usando SIMD (AVX2/AVX-512).

Vigilância Ativa: Monitorar a integridade do kernel host através de rastreamento de hardware, impedindo a persistência de malwares avançados.

Stack Tecnológica
Core Language: Assembly x86-64 (NASM/Yasm) para o kernel do hypervisor.


System Language: C++20 para wrappers de gerenciamento e inicialização de alto nível.

O Micro-Hypervisor (O Escudo)
O quê: Um monitor de máquina virtual (VMM) minimalista.

Como: Configuraremos o bit VMXE no registrador CR4, inicializaremos a região VMXON e montaremos o VMCS (Virtual Machine Control Structure).

Por quê: Para criar um anel de privilégio superior ao Kernel (Ring -1), onde o SO host não pode ler nem escrever na memória do enclave criptográfico.

Engine de Marshalling Binário (O Fluxo)
O quê: Um sistema de transporte de dados ultra-rápido entre o SO e o Enclave.

Como: Utilizaremos registradores de 256 ou 512 bits para empacotar múltiplos blocos de dados simultaneamente.

Por quê: O marshalling tradicional via software é o gargalo de sistemas seguros. O uso de SIMD permite processar dados em paralelo, garantindo que a segurança não destrua o throughput do sistema.

Rootkit Detector via Hardware-Trace (A Vigilância)
O quê: Um auditor de integridade em tempo real.

Como: Configuraremos o hypervisor para gerar um VMExit sempre que houver uma tentativa de escrita em registradores sensíveis (como LSTAR para syscalls ou o IDTR para interrupções).

Por quê: Rootkits modernos operam no Ring 0. Ao monitorar via hardware as transições de estado, detectamos anomalias antes mesmo de o malwares conseguir mascarar sua presença.

Objetivo Final
O objetivo é entregar uma prova de conceito funcional onde:

O sistema operacional é "promovido" a Guest.

Uma aplicação C++ envia dados para serem criptografados no Enclave.

O Marshalling ocorre em ciclos de CPU recordes.

Qualquer tentativa de um vírus de nível de kernel de interceptar a syscall é barrada e logada pelo Hypervisor.
